Проект: Шахматы на двух людей
Общие действия.
Необходимо прописать классы фигур для каждого из типов фигур, реализовать запись ходов, чтобы можно было нажать на каждый ход и посмотреть
позицию в тот момент времени, возможно впоследствии создать небольшую базу данных прошедших партий. 
Реализовать функцию, чтобы при передаче хода доска переворачивалась на экране. При съедании фигуры нужно добавлять ее в съеденные фигуры, 
каждой фигуре присвоить свою ценность и на основе съеденных фигур показывать преимущество одного игрока над другим по материалу на доске.
План по шагам.
1. Создание шахматной доски, объектов на ней и добавление возможности хода каждой фигурой с учетом ее особенностей.
2. Создание возможности хода для нестандартных случаев (таких как взятие на проходе) и блокировка хода при некоторых ситуациях (рокировка во время шаха).
3. Создание условий победы.
4. Переворачивание доски при смене хода.
5. Запись каждого хода в отдельном окне и создание базы данных по прошедшим партиям.
6. Присваивание каждой фигуре свою определенную ценность, вывод материального преимущества в течении партии.
